## 1. 專案摘要 (Executive Summary)

本報告旨在詳細闡述「藍隊防禦系統」專案的完整開發歷程、核心系統架構、關鍵功能以及最終的安全性驗證結果。

本專案成功建構了一個多層次、具備智慧分析能力的資訊安全監控與防禦原型系統。系統的核心由**Web 應用程式防火牆 (WAF)** 與 **安全資訊與事件管理系統 (SIEM)** 兩大防禦模組構成，並輔以一套完全自動化的**「紅隊即程式碼 (Red Team as Code)」攻擊模擬框架**，用以科學化地驗證系統的防禦成效。

在開發過程中，我們採用了測試導向的開發模式，針對 MITRE ATT&CK 框架中的多種關鍵威脅技術，逐一開發偵測規則並立即進行模擬攻擊驗證。最終測試結果表明，本系統不僅能有效偵測及攔截多種類型的網路攻擊與端點威脅，其最關鍵的**狀態化分析**與**關聯分析**能力，更能成功識別出如勒索軟體、多階段攻擊鏈等複雜的現代資安威脅。

## 2. 開發歷程回顧 (Development Process)

本專案的開發過程是一個典型的敏捷迭代、由淺入深的過程，大致可分為以下六個階段：

#### **階段一：環境初始化與策略轉向 (Initial Setup & Strategic Pivot)**

專案初期，我們嘗試使用業界標準的 Docker 及 WSL (Windows Subsystem for Linux) 來部署 Nginx + ModSecurity 環境。然而，在用戶端環境中遭遇了數次 Docker 服務啟動失敗與 WSL 初始化不穩定的問題。為了不讓環境問題阻礙核心功能的開發，我們果斷地進行了**策略轉向 (Pivot)**，決定採用純 Python 實現一個輕量級的 WAF 反向代理與後端模擬伺服器。此決策確保了開發環境的穩定性與跨平台兼容性，是專案得以順利推進的關鍵。

#### **階段二：核心防禦模組建構 (Core Modules Implementation)**

在確定技術棧後，我們著手建構了兩大核心模組：
1.  **`waf_proxy.py`:** 利用 Python 內建函式庫，快速實現了一個具備基礎反向代理與規則匹配功能的 WAF。
2.  **`siem_dashboards.py`:** 設計並實現了一個物件導向的 SIEM 系統，定義了 `SIEMRule` 與 `SOCDashboard` 類別，為後續的規則擴充奠定了彈性的基礎架構。

#### **階段三：基礎攻擊模擬與驗證 (Basic ATT&CK Simulation)**

在核心框架搭建完畢後，我們立即進入了第一個「開發-測試」循環。我們選定了幾種最基礎、最常見的駭客技術，並採用「測試導向」的模式進行開發：
1.  **定義威脅：** 建立如 `test_atomic_t1059_003.py` (指令注入) 等攻擊腳本。
2.  **開發防禦：** 在 SIEM 中加入對應的偵測規則 (如 R005)。
3.  **立即驗證：** 執行攻擊腳本，確認警報是否如預期般觸發。
此階段成功驗證了系統對**指令注入 (T1059.003)**、**憑證竊取 (T1003.001)**、**C2 通訊 (T1071.001)** 與 **釣魚郵件 (T1566.001)** 的基礎偵測能力。

#### **階段四：進階威脅偵測能力深化 (Advanced Threat Detection)**

在基礎防禦穩固後，我們開始挑戰更複雜的威脅情境：
1.  **勒索軟體 (T1486):** 為了偵測此類「行為模式」，我們為 SIEM 引入了**狀態化分析 (Stateful Analysis)** 的能力，使其能追蹤在短時間內的大量檔案操作。
2.  **持久化 (T1053.005):** 透過監控 `schtasks.exe` 的使用，賦予系統偵測駭客建立後門行為的能力。
3.  **橫向移動 (T1021.002):** 透過監控對管理員共用的存取，讓系統具備了識別攻擊在內網擴散的能力。

#### **階段五：WAF 硬化與壓力測試 (WAF Hardening & Stress Testing)**

此階段我們將焦點轉回系統的對外接口。我們強化了 WAF 的規則庫，使其能抵禦**時間盲注 SQLi**、**進階 XSS** 與 **SSRF** 等更刁鑽的 Web 攻擊。隨後，我們開發了 `test_waf_stress.py` 腳本，對 WAF 進行了全面的自動化壓力測試，確保了其防護的有效性。

#### **階段六：攻擊鏈關聯分析 (Attack Chain Correlation)**

這是專案的最高潮。我們為 SIEM 系統引入了最能體現其智慧的**關聯分析引擎**。透過新增一條特殊的關聯規則 (C001)，系統被賦予了自動將來自單一攻擊者的多個獨立警報串聯起來，從而識別出**完整攻擊鏈**的能力。最終，我們透過 `test_attack_chain.py` 腳本成功驗證了此項終極功能。

## 3. 系統架構與功能詳解

*(此章節詳細內容請參考前次對談中的「更詳細的介紹」，此處簡述核心)*

#### **3.1 Web 應用程式防火牆 (`waf_proxy.py`)**
- **角色:** 系統的網路閘道，保護後端應用。
- **技術核心:** 基於正規表示式 (Regex) 的規則引擎，對經過 URL 解碼的 HTTP 請求進行深度內容檢測。
- **關鍵功能:** SQLi, XSS, SSRF, DoS 防護。

#### **3.2 安全資訊與事件管理系統 (`siem_dashboards.py`)**
- **角色:** 系統的分析大腦，處理並分析日誌事件。
- **技術核心:** 物件導向的事件處理器，包含一個可擴充的規則庫。
- **關鍵功能:**
    - **廣泛的 ATT&CK 技術偵測。**
    - **狀態化分析：** 透過內部追蹤器，偵測如勒索軟體等基於「行為模式」的威脅。
    - **關聯分析：** 將獨立警報自動關聯成高嚴重性的「多階段攻擊」事件。

#### **3.3 自動化攻擊模擬框架 (所有 `test_*.py`)**
- **角色:** 系統的品質保證與自動化紅隊。
- **技術核心:** 採用「日誌偽造」技術，以安全、穩定、快速的方式模擬駭客攻擊。
- **關鍵功能:** 為每一條防禦規則提供可重複執行的驗證案例，並自動產出包含證據的測試報告。

## 5. 結論與未來展望

本專案成功地從無到有，建構並驗證了一個功能強大且設計精良的模擬資安防禦系統。它不僅證明了在端點與網路層面上的廣泛威脅偵測能力，更透過其獨特的狀態化與關聯分析引擎，展現了洞察複雜攻擊的智慧。

**未來可行的發展方向包括：**
1.  **整合真實日誌源：** 接入如 Windows 事件日誌、Sysmon、真實的防火牆或 Web 伺服器日誌，以在真實環境中驗證其效果。
2.  **擴充規則庫：** 持續關注最新的威脅情資，並將新的攻擊手法轉換為新的偵測規則。
3.  **開發視覺化儀表板：** 為 SIEM 系統開發一個圖形化使用者介面 (GUI)，將警報與數據以更直觀的方式呈現給資安分析師。
4.  **自動化應對 (SOAR)：** 在現有基礎上，加入自動化應對的劇本 (Playbook)，例如在偵測到橫向移動時，自動執行腳本隔離受駭主機的網路。
