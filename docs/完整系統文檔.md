## 系統概述

### 🎯 系統目標

建立一個具備**國防等級**安全防護的 Web 管理系統，實現：
- ✅ 雙角色（Admin/User）權限管理
- ✅ 密碼安全輸入（隱藏顯示）
- ✅ 機密資料管理
- ✅ 中央伺服器資料傳輸
- ✅ 多層攻擊防護（SQL注入、XSS、路徑遍歷、命令注入）
- ✅ DDoS 防護（速率限制 + IP 封鎖）
- ✅ 即時攻擊監控和報告
- ✅ SHA-256 加密傳輸證明

| 項目 | 規格 |

| 開發語言 | Python 3.8+ |

### 📡 中央伺服器傳輸

#### 傳輸流程
```
Web 系統事件發生
    ↓
1. 準備資料
   {type: "LOGIN_EVENT", user: "admin", ...}
    ↓
2. SHA-256 加密
   data + ENCRYPTION_KEY → 加密結果
    ↓
3. MD5 校驗和
   data → checksum
    ↓
4. 構建傳輸包
   {id, timestamp, user, data_type, encrypted_data, checksum, status}
    ↓
5. POST 到中央伺服器
   POST http://127.0.0.1:9000/api/receive
   Header: X-API-Key
    ↓
6. 中央伺服器驗證
   - API Key 驗證
   - 校驗和驗證
   - 儲存記錄
    ↓
7. 返回確認
   {status: "success", received_id: "..."}
    ↓
8. Web 系統記錄結果
   transmission["status"] = "SUCCESS"
```

#### SHA-256 加密證明

**加密演算法**:
```python
def _encrypt_data(self, data):
    """SHA-256 加密證明"""
    # 1. 將資料序列化為 JSON 字符串
    data_str = json.dumps(data, ensure_ascii=False)
    
    # 2. 使用 SHA-256 加密：資料 + 密鑰
    encrypted = hashlib.sha256(
        data_str.encode() + SecurityConfig.ENCRYPTION_KEY
    ).hexdigest()
    
    # 3. 返回 64 字符十六進制字符串
    return encrypted
```

**加密證明示例**:
```
原始資料:
{
  "username": "admin",
  "role": "admin",
  "ip": "127.0.0.1"
}

SHA-256 加密後:
e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855

特性:
✓ 固定長度 64 字符
✓ 不可逆（無法從加密結果還原原始資料）
✓ 雪崩效應（資料微小變化導致加密結果巨大變化）
✓ 唯一性（相同資料和密鑰始終產生相同結果）
✓ 抗碰撞（幾乎不可能找到兩個不同資料產生相同結果）
```

**校驗和證明**:
```python
def _calculate_checksum(self, data):
    """MD5 校驗和證明"""
    # 1. 將資料排序後序列化（確保一致性）
    data_str = json.dumps(data, sort_keys=True, ensure_ascii=False)
    
    # 2. 使用 MD5 計算校驗和
    checksum = hashlib.md5(data_str.encode()).hexdigest()
    
    # 3. 返回 32 字符十六進制字符串
    return checksum
```

**完整傳輸包示例**:
```json
{
  "id": "a3f2e1d4c5b6a7",
  "timestamp": "2025-10-11T10:30:25.123456",
  "user": "admin",
  "data_type": "LOGIN_EVENT",
  "data": "e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855",
  "checksum": "5d41402abc4b2a76b9719d911017c592",
  "status": "SUCCESS"
}
```

**證明方式**:
1. **查看傳輸日誌**: 儀表板中顯示 SHA-256 加密結果的前 16 字符
2. **查看中央伺服器**: 訪問 `http://127.0.0.1:9000/api/transmissions` 查看完整加密資料
3. **驗證完整性**: 使用 checksum 驗證資料未被篡改

## 安全機制

### 🔒 密碼安全

#### 1. 密碼雜湊
```python
# PBKDF2-HMAC-SHA256
password_hash = hashlib.pbkdf2_hmac(
    'sha256',                    # 使用 SHA-256
    password.encode('utf-8'),    # 用戶密碼
    SALT.encode('utf-8'),        # 隨機鹽值（32 字節）
    100000                       # 100,000 次迭代
).hex()
```

**安全特性**:
- ✅ 使用業界標準 PBKDF2 算法
- ✅ SHA-256 作為基礎雜湊函數
- ✅ 隨機鹽值防止彩虹表攻擊
- ✅ 100,000 次迭代延緩暴力破解
- ✅ 不可逆（無法還原原始密碼）

#### 2. 密碼輸入框
```html
<input type="password" 
       id="password" 
       name="password" 
       required 
       autocomplete="current-password" 
       placeholder="請輸入密碼">
```

**特性**:
- ✅ `type="password"` - 輸入顯示為 `***`
- ✅ `autocomplete` - 支援密碼管理器
- ✅ `required` - 必填驗證

#### 3. 帳號鎖定
```python
# 3 次失敗嘗試
if len(login_attempts[username]) >= 3:
    locked_accounts[username] = current_time + 300  # 鎖定 5 分鐘
    log_attack("BRUTE_FORCE", username)
    transmit_alert("BRUTE_FORCE", username)
```

**防護機制**:
- ✅ 失敗 3 次自動鎖定
- ✅ 鎖定時間 300 秒（5 分鐘）
- ✅ 記錄攻擊事件
- ✅ 傳輸安全警報

### 🔐 CSRF 保護

#### Token 生成
```python
def generate_csrf_token(session_id):
    token = secrets.token_urlsafe(32)  # 32 字節隨機 Token
    csrf_tokens[session_id] = {
        "token": token,
        "created": time.time()
    }
    return token
```

#### Token 驗證
```python
def validate_csrf_token(session_id, token):
    # 1. 檢查 Token 是否存在
    if session_id not in csrf_tokens:
        return False
    
    # 2. 檢查過期（10 分鐘）
    if time.time() - csrf_tokens[session_id]["created"] > 600:
        del csrf_tokens[session_id]
        return False
    
    # 3. 使用 hmac.compare_digest 防止時序攻擊
    return hmac.compare_digest(
        csrf_tokens[session_id]["token"], 
        token
    )
```

#### 使用流程
```
1. GET /login → 生成 CSRF Token → 嵌入表單
2. 用戶填寫表單
3. POST /login → 附帶 CSRF Token
4. 伺服器驗證 Token → 通過才處理請求
```
