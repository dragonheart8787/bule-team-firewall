## 🔐 安全功能驗證

### 1. 密碼輸入框 ✅

**驗證方式：**
1. 開啟瀏覽器訪問 http://127.0.0.1:5000
2. 在密碼欄輸入密碼
3. 觀察密碼顯示為 `**********`

**技術實現：**
```html
<input type="password" name="password">
```

### 2. 中央伺服器資料傳輸 ✅

**驗證方式：**
1. 以 admin 登入 Web 系統
2. 執行任何操作
3. 查看「中央伺服器傳輸記錄」
4. 或訪問 http://127.0.0.1:9000

**傳輸內容：**
- 登入/登出事件
- 資料訪問事件
- 資料修改事件
- 安全警報事件
- IP 管理事件

**加密方式：**
- SHA-256 加密
- MD5 校驗和
- 時間戳驗證

### 3. 中間層攻擊防護 ✅

**防護類型：**
- SQL 注入：檢測 union, select, drop, insert...
- XSS 攻擊：檢測 &lt;script&gt;, javascript:...
- 路徑遍歷：檢測 ../, ..\
- 命令注入：檢測 |, ;, `, $...

**防護流程：**
```
請求 → 攻擊檢測 → 發現攻擊？
                     ↓ 是
                  阻擋 (403) → 記錄 → 傳輸警報
```

### 4. DDoS 防護 ✅

**防護機制：**
- 全局限制：100 req/min
- 端點限制：20 req/min
- 自動 IP 封鎖
- 威脅等級調整

**觸發條件：**
- 超過速率限制 → 返回 429
- 持續超過 → 封鎖 IP
- 多次攻擊 → 提升威脅等級

### 5. APT 檢測 ✅

**檢測模式：**
- SQL 關鍵字
- 腳本標籤
- 系統命令
- 路徑遍歷
- 特殊字符組合

**檢測後動作：**
1. 立即阻擋請求
2. 記錄攻擊模式
3. 傳輸到中央伺服器
4. 提升威脅等級
5. 通知管理員

## 🔍 故障排除

### 問題 1: 中央伺服器無法連接

**症狀**: 傳輸狀態顯示 "OFFLINE"

**解決方法**:
```bash
1. 檢查中央伺服器是否運行
2. 訪問 http://127.0.0.1:9000/health
3. 如果失敗，重新啟動中央伺服器
```

### 問題 2: 測試失敗

**症狀**: 測試顯示 [FAIL]

**解決方法**:
```bash
1. 確認兩個服務都在運行
2. 等待 15 秒讓服務完全啟動
3. 重新運行測試
```

### 問題 3: 端口被占用

**症狀**: "Address already in use"

**解決方法**:
```bash
# 查找占用端口的進程
netstat -ano | findstr :5000
netstat -ano | findstr :9000

# 停止進程
taskkill /f /im python.exe
```
