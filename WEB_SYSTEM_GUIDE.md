## 🛡️ 安全防護詳解

### 1. 中間層攻擊防護

#### 檢測的攻擊類型

| 攻擊類型 | 檢測模式 | 防護動作 |
|---------|---------|---------|
| **SQL 注入** | union, select, insert, update, delete, drop, --, ; | 阻止請求 + 記錄 + 通知 |
| **XSS 攻擊** | &lt;script, javascript:, onerror=, onload= | 阻止請求 + 記錄 + 通知 |
| **路徑遍歷** | ../, ..\ | 阻止請求 + 記錄 + 通知 |
| **命令注入** | \|, ;, \`, $, &&, \|\| | 阻止請求 + 記錄 + 通知 |

#### 防護流程

```
用戶請求
    ↓
DDoS 檢查
    ↓
速率限制檢查
    ↓
中間層攻擊檢查
    ↓ (通過)
處理請求
    ↓
傳輸到中央伺服器
    ↓
返回結果
```

### 2. DDoS 防護

#### 多層次防護

```python
# 全局速率限制
DDOS_THRESHOLD = 100  # 每分鐘 100 次請求

# 端點速率限制
RATE_LIMIT = 20  # 每個端點每分鐘 20 次

# 時間窗口
DDOS_WINDOW = 60  # 60 秒
```

#### 觸發條件

- **條件 1**: 60 秒內總請求數 > 100 次
- **條件 2**: 60 秒內單端點請求數 > 20 次

#### 觸發後動作

1. ✅ 立即封鎖攻擊者 IP
2. ✅ 記錄到攻擊日誌
3. ✅ 傳輸警報到中央伺服器
4. ✅ 提升威脅等級
5. ✅ 管理員可查看並解封

### 3. CSRF 保護

#### 工作原理

```
1. 用戶訪問登入頁面
   ↓
2. 伺服器生成 CSRF Token
   ↓
3. Token 嵌入到表單中
   ↓
4. 用戶提交表單
   ↓
5. 伺服器驗證 Token
   ↓ (匹配)
6. 處理請求
```

#### Token 特性

- **生成**: 每次頁面載入生成新 Token
- **長度**: 32 字節 URL-safe
- **有效期**: 10 分鐘
- **驗證**: 時間恆定比對（防時序攻擊）

### 4. Session 安全管理

#### Session 機制

```python
# Session 配置
SESSION_TIMEOUT = 1800  # 30 分鐘無活動自動登出

# Session 內容
session = {
    'user': 'admin',
    'role': 'admin',
    'login_time': 1234567890
}
```

#### 安全特性

- ✅ **加密存儲**: Flask 內建加密 Session
- ✅ **自動過期**: 30 分鐘無活動自動清除
- ✅ **防劫持**: 使用 HttpOnly Cookie
- ✅ **防 CSRF**: Token 綁定 Session

## 📡 中央伺服器資料傳輸

### 傳輸的事件類型

| 事件類型 | 觸發時機 | 包含資料 |
|---------|---------|---------|
| `LOGIN_EVENT` | 用戶登入成功 | 用戶名、角色、IP |
| `LOGOUT_EVENT` | 用戶登出 | 用戶名 |
| `DATA_ACCESS` | 查看機密資料 | 角色、訪問的資料類別 |
| `DATA_MODIFICATION` | 修改資料 | 類別、鍵名 |
| `SECURITY_ALERT` | 檢測到攻擊 | 攻擊類型、IP、用戶名 |
| `IP_UNBLOCKED` | 解封 IP | IP 地址 |

### 傳輸流程

```
事件觸發
    ↓
資料加密 (SHA-256 + 密鑰)
    ↓
計算校驗和 (MD5)
    ↓
建立傳輸記錄
    ↓
模擬發送到中央伺服器
    ↓
記錄傳輸日誌
    ↓
管理員可查看
```

### 傳輸記錄格式

```json
{
  "id": "a1b2c3d4e5f6...",
  "timestamp": "2025-10-10T14:30:00.123456",
  "user": "admin",
  "data_type": "LOGIN_EVENT",
  "data": "encrypted_hash_here",
  "checksum": "md5_checksum_here",
  "status": "SUCCESS"
}
```

## 🚨 威脅等級系統

### 自動調整規則

| 等級 | 圖標 | 條件 | 說明 |
|------|------|------|------|
| **GREEN** | 🟢 | 0-1 攻擊/5分鐘 | 系統正常運行，無檢測到威脅 |
| **YELLOW** | 🟡 | 2-4 攻擊/5分鐘 | 檢測到輕微可疑活動，保持警戒 |
| **ORANGE** | 🟠 | 5-9 攻擊/5分鐘 | 檢測到多次攻擊嘗試，已加強防護 |
| **RED** | 🔴 | 10+ 攻擊/5分鐘 | 遭受嚴重攻擊，所有防護系統已啟動 |

### 等級影響

- **視覺反饋**: 頁面頂部顯示不同顏色
- **警報級別**: 傳輸到中央伺服器的優先級
- **防護強度**: 自動調整防護參數

## 🎯 測試攻擊場景

### 測試 1: 暴力破解

```
步驟:
1. 訪問 http://127.0.0.1:5000
2. 輸入用戶名: admin
3. 輸入錯誤密碼 3 次
4. 觀察帳號鎖定提示
5. 以 admin 登入查看攻擊記錄
```

### 測試 2: SQL 注入

```
步驟:
1. 在登入頁面用戶名輸入: admin' OR '1'='1
2. 觀察「檢測到可疑請求」錯誤
3. 以 admin 登入
4. 查看攻擊記錄（顯示 MIDDLEWARE_ATTACK）
```

### 測試 3: XSS 攻擊

```
步驟:
1. 在登入頁面用戶名輸入: <script>alert(1)</script>
2. 觀察「檢測到可疑請求」錯誤
3. 以 admin 登入
4. 查看攻擊記錄
```

### 測試 4: DDoS 攻擊

```
步驟:
1. 使用腳本快速發送 > 100 次請求
2. 觀察「請求過於頻繁」錯誤
3. IP 被自動封鎖
4. 以 admin 登入查看封鎖列表
```

### 測試 5: 中央伺服器傳輸

```
步驟:
1. 以 admin 登入
2. 執行各種操作（查看資料、修改資料等）
3. 查看「中央伺服器傳輸記錄」
4. 觀察所有事件都已記錄
```
